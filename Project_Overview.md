# Project Overview

## Introduction

This project demonstrates my approach to investigating and documenting software defects through structured manual testing. Using the SauceDemo application, I evaluated the checkout workflow, reproduced application behavior, documented observations, and prepared professional QA deliverables.

The objective was not only to identify defects but also to communicate findings clearly through test cases, execution reports, screenshots, and bug documentation that developers can easily understand and reproduce.

---

## Objectives

- Evaluate the checkout process through manual testing.
- Design and execute structured test cases.
- Identify functional defects.
- Reproduce software issues consistently.
- Document findings using industry-standard QA practices.

---

## Scope

The testing focused on the checkout workflow, including customer information validation and order completion.

---

## Deliverables

- Test Case Design
- Test Execution Report
- Bug Report
- Supporting Screenshots
- Testing Notes

---

## Tools Used

- SauceDemo
- Microsoft Excel
- Microsoft Word
- Git
- GitHub

---

## Outcome

The project successfully identified a validation issue within the checkout process and demonstrated my ability to investigate, reproduce, document, and communicate software defects using structured QA documentation.
